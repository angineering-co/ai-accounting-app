# 營業稅結算分錄範例（給會計師確認）

本文件說明期別申報結案時，系統自動產生的「營業稅結算分錄」長相，供會計師確認帳務處理是否正確。

## 這筆分錄在做什麼

每期（401 兩個月一期）申報時，系統會把當期累積在以下兩個科目的稅額沖結，並把淨額導向應付或留抵：

| 代碼 | 科目 | 期間餘額性質 |
| --- | --- | --- |
| 2134 | 銷項稅額 | 當期銷項稅額（貸方累積） |
| 1144 | 進項稅額 | 當期得扣抵進項稅額（借方累積） |
| 2132 | 應付稅捐 | 本期應實繳營業稅（貸方，待繳款後沖銷） |
| 1145 | 留抵稅額 | 累積留抵（借方，逐期結轉） |

- **傳票類型**：轉帳（無現金科目；本筆只做稅額重分類，實際繳款另開分錄）。
- **狀態**：草稿。由承辦人於一般過帳流程過帳，過帳後才賦予傳票號。
- **日期**：該期末月最後一日（例如 113 年 1-2 月期 → 113/02/28）。

## 數字來源

三個輸入皆取自當期 .TET_U 申報書：

| 輸入 | 申報書欄位 | 說明 |
| --- | --- | --- |
| 本期銷項稅額 | Field 82 | 本期(月)銷項稅額合計 |
| 得扣抵進項稅額 | Field 87 | 得扣抵進項稅額合計（含固定資產） |
| 上期累積留抵 | Field 88 | 上期(月)累積留抵稅額 |

## 計算邏輯

```
本期應繳    = max(0, 銷項稅額 − 進項稅額 − 上期留抵)        → 貸 2132 應付稅捐
期末留抵    = max(0, 進項稅額 + 上期留抵 − 銷項稅額)        → 1145 留抵稅額期末應有餘額
留抵增減    = 期末留抵 − 上期留抵                          → 增加借記 1145、減少貸記 1145
```

其中「期末留抵」是 1145 留抵稅額在期末應有的餘額；實際過帳的是「留抵增減」（期末留抵與上期留抵的差額），增加時借記 1145、減少時貸記 1145。

借貸恆等於「銷項稅額」與「進項稅額」沖平，故任何情境皆平衡。

## 範例

### 範例 1：本期應繳，無上期留抵（3 行）

當期銷項稅額 50,000、進項稅額 30,000、上期留抵 0。

| 科目 | 借方 | 貸方 |
| --- | ---: | ---: |
| 2134 銷項稅額 | 50,000 | |
| 1144 進項稅額 | | 30,000 |
| 2132 應付稅捐 | | 20,000 |
| **合計** | **50,000** | **50,000** |

本期應繳營業稅 20,000。

### 範例 2：本期應繳，且沖抵上期留抵（4 行）

當期銷項稅額 50,000、進項稅額 30,000、上期留抵 8,000。

| 科目 | 借方 | 貸方 |
| --- | ---: | ---: |
| 2134 銷項稅額 | 50,000 | |
| 1144 進項稅額 | | 30,000 |
| 1145 留抵稅額 | | 8,000 |
| 2132 應付稅捐 | | 12,000 |
| **合計** | **50,000** | **50,000** |

先以上期留抵 8,000 全數抵減，本期實繳 12,000，期末累積留抵歸零。

### 範例 3：本期留抵（進項大於銷項，無上期留抵）（3 行）

當期銷項稅額 20,000、進項稅額 30,000、上期留抵 0。

| 科目 | 借方 | 貸方 |
| --- | ---: | ---: |
| 2134 銷項稅額 | 20,000 | |
| 1145 留抵稅額 | 10,000 | |
| 1144 進項稅額 | | 30,000 |
| **合計** | **30,000** | **30,000** |

本期新增留抵 10,000，期末累積留抵 10,000，本期不需繳稅。

### 範例 4：銷項大於進項，但加計上期留抵仍為留抵（部分沖抵上期留抵）（3 行）

當期銷項稅額 25,000、進項稅額 20,000、上期留抵 10,000。

| 科目 | 借方 | 貸方 |
| --- | ---: | ---: |
| 2134 銷項稅額 | 25,000 | |
| 1144 進項稅額 | | 20,000 |
| 1145 留抵稅額 | | 5,000 |
| **合計** | **25,000** | **25,000** |

本期消耗上期留抵 5,000（10,000 → 5,000），期末累積留抵剩 5,000，本期不需繳稅。

### 範例 5：銷項等於進項，有上期留抵（2 行，留抵不變）

當期銷項稅額 10,000、進項稅額 10,000、上期留抵 4,000。

| 科目 | 借方 | 貸方 |
| --- | ---: | ---: |
| 2134 銷項稅額 | 10,000 | |
| 1144 進項稅額 | | 10,000 |
| **合計** | **10,000** | **10,000** |

本期銷進相抵，無應繳亦無留抵變動，上期留抵 4,000 原封結轉，不動 1145。

### 範例 6：停業或無交易，僅有上期留抵（不產生分錄）

當期銷項稅額 0、進項稅額 0、上期留抵 5,000。

當期沒有任何銷項與進項，留抵原封結轉，系統不會產生分錄（1145 餘額已反映上期結轉，本期無需入帳）。

## 留抵如何結轉到下一期

期末留抵不另存一個「餘額欄位」，而是同時由兩個地方記錄，下一期申報時自動帶入：

1. **1145 留抵稅額分類帳餘額**：由於每期只過帳「留抵增減」（差額），1145 的累計餘額（所有已過帳借方減貸方）即等於當下的累積留抵。這是帳上的真實餘額，隨分類帳即時計算。
2. **每期申報快照**：產生 .TET_U 時，系統會把該期申報書的累積留抵（Field 95 本期累積留抵稅額）存進該期別紀錄。這是「已申報」的留抵數字。

下一期需要填上期累積留抵（Field 88）時，依序取用：

1. 上一期的申報快照（Field 95）。優先採用，因為要與上期實際申報的數字一致（國稅局逐期勾稽），且不受結算分錄是否已過帳影響。
2. 若無快照（例如年中才轉入、無上期申報紀錄），改讀 1145 已過帳餘額（需先由開帳分錄建立期初留抵）。
3. 皆無則為 0，由承辦人手動填寫。

### 跨期範例

- 第 N 期：期末留抵 10,000 → 結算分錄過帳「借 1145 10,000」→ 1145 分類帳餘額 10,000；申報快照 Field 95 = 10,000。
- 第 N+1 期：本期應繳並沖抵留抵 → 結算分錄過帳「貸 1145 10,000」→ 1145 分類帳餘額回到 0；Field 88 自上一期快照帶入 10,000。

一般情況下兩者一致。

### 對帳提醒

下列情況下，申報快照（Field 95）與 1145 分類帳餘額可能暫時不一致：

- **結算分錄尚未過帳（草稿）**：分類帳餘額會落後一期，快照已有數字。這也是優先採用快照的原因。
- **有應退稅額（Field 94）**：結算分錄把整筆併入留抵（Field 92），快照存的是扣除應退後的 Field 95，差額即應退稅額（目前尚未另立應收退稅款分錄）。
- **1145 有手動調整分錄**：帳上餘額變動但未反映在任何一期的 Field 95。

這些差異本身是有用的訊號，代表有待過帳的分錄、未入帳的退稅、或人工調整需檢視，後續會以對帳檢查主動標示。

## 目前不涵蓋（待確認是否需要）

- **應退稅額（申報書 Field 94）**：例如零稅率或固定資產產生的得退稅額。目前一律併入留抵（1145），不另立應收退稅款科目。
- **中途歇業 / 年底調整**（Field 85 補徵、Field 89 應退）：未納入本分錄。
- **實際繳款分錄**：繳納營業稅時的「借 2132／貸 銀行存款」目前需由承辦人依銀行對帳手動入帳，尚未自動產生。
